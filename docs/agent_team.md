# 多智能体系统扩展 - 详细设计文档

## 一、 设置面板菜单修改

### 1. 菜单结构调整
- 在现有的"智能体"菜单下方，新增一个一级菜单项：**"智能小组"**
- 点击"智能小组"后，右侧主内容区切换为"智能小组"管理面板

### 2. 智能小组面板设计
- **布局**：面板顶部右侧为 **"创建小组"** 按钮，下方为小组列表区域
- **列表字段**：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| **小组名称** | 用户创建时定义的名称 | 孪生城市生产小组 |
| **组长名称** | 被指定为组长的智能体名字 | 架构师-元宝 |
| **成员名称** | 该小组所有成员的名字，以"、"分隔 | 架构师-元宝、设计师-CodeX、工程师-DEV |
| **进度** | 表示小组任务的整体进度状态 | `进行中` / `已完成` / `等待中` / `75%` |

### 3. 创建小组功能流程
- **触发**：点击"创建小组"按钮，弹出模态窗口
- **面板内容**：
  - **a. 选择组长**：下拉选择器，数据源来自"智能体"菜单中已存在的智能体列表（必须选择）
  - **b. 输入任务**：多行文本输入框，描述小组需要完成的最终目标
  - **按钮**：底部有"取消"和"创建"两个按钮
- **创建逻辑**：
  1. 用户点击"创建"，前端发送 `{组长ID/Name, 任务描述}` 到后端
  2. 后端调用**组长智能体**服务，将任务描述与**LLM**进行结构化对话
  3. **组长智能体**收到LLM返回的规划结果后，调用 `agent_service.createDynamicAgents(planning_result)`
  4. `agent_service` 根据规划动态创建成员智能体，为每个新智能体初始化 `task_queue`
  5. 后端创建"智能小组"记录，状态设为"进行中"，关联组长和成员
  6. 后端推送消息，通知前端更新列表

## 二、 聊天面板重构设计

### 1. 整体布局
- 将单一聊天面板拆分为**左右并列的两个区域**，中间有可拖动分割条
- **左侧区域（小组协作区）**：标题为"**小组协作**"，副标题显示当前活跃的小组名称
- **右侧区域（私人对话区）**：标题为"**与[智能体名]对话**"，功能与原聊天面板一致

### 2. 左侧区域（小组协作区）逻辑
- **内容**：实时展示**当前所选智能小组**内部成员间的对话
- **消息流**：
  1. 小组创建后，**组长**在组内频道发布任务，@相关成员
  2. 被分配任务的成员在**队列空闲时**，在组内频道回复处理进度
  3. 完成任务的成员@组长回复结果
- **关键规则**：**用户发言、用户与智能体的私人对话，绝对不在此区域显示**。此区域是纯"NPC"后台讨论区

### 3. 右侧区域（私人对话区）逻辑
- **功能**：用户可从顶部智能体选择器中切换要与哪个智能体进行一对一对话
- **任务队列交互规则**：
  - 每个智能体维护一个 **`task_queue`**
  - 用户与智能体A私聊时，**组长**向A下达的小组任务会**异步、有序**插入A的 `task_queue` 末尾
  - **执行规则**：
    1. 智能体A**按顺序**处理 `task_queue` 中的任务
    2. 处理"小组任务"时，对话目标依然是**组长**，在**左侧区域**交互
    3. 完成"小组任务"后，必须将结果发送回**组长**
    4. 用户与智能体A的私聊**不会被打断**，除非任务需要用户输入

### 4. 记忆与存储
- **两类会话独立存储**：
  - `session_type: 'group_chat'` - 存储**左侧区域**的小组内部对话，关联 `group_id`
  - `session_type: 'private_chat'` - 存储**右侧区域**的用户-智能体一对一对话，关联 `user_id` 和 `agent_id`
- **会话管理**：在"设置面板"的"会话管理"中，提供筛选查看"私人对话"和"小组协作"历史

## 三、 后端与数据流关键点

### 1. 消息路由
- `Channel_Group`: 小组内部通信，广播给所有成员，存入`group_chat`记忆
- `Channel_Private`: 用户与智能体私聊，点对点，存入`private_chat`记忆

### 2. 事件驱动
- 智能体完成任务、队列状态变更等，通过事件通知机制（如WebSocket）推动前端实时更新

### 3. Agent Service扩展
- `createTeam(leaderId, mission)`: 创建小组入口
- `createDynamicAgents(rolePlan)`: 根据规划实例化角色智能体
- `assignTaskToAgent(agentId, task, fromLeaderId)`: 组长给组员派发任务

### 4. 前端状态管理
- 全局管理"当前活跃的智能小组"状态，确保左侧面板显示正确内容

## 四、 API接口示例

```javascript
// 创建智能小组
POST /api/teams/create
Body: {
  "leader_id": "agent_123",
  "mission": "开发一个孪生城市管理系统",
  "team_name": "孪生城市生产小组"
}

// 获取小组列表
GET /api/teams

// 获取小组详情
GET /api/teams/:teamId

// 小组内部消息
POST /api/teams/:teamId/messages
Body: {
  "sender_id": "agent_123",
  "content": "任务分配：@agent_456 请负责数据库设计",
  "type": "assignment"
}

// 获取小组聊天记录
GET /api/teams/:teamId/chat-history
```

## 五、 数据结构

```javascript
// 智能小组
{
  id: "team_001",
  name: "孪生城市生产小组",
  leader_id: "agent_001",
  mission: "开发孪生城市管理系统",
  status: "running", // running, completed, paused
  progress: 0.3,
  created_at: "2024-01-15T10:00:00Z",
  members: [
    { agent_id: "agent_001", role: "组长" },
    { agent_id: "agent_002", role: "架构师" },
    { agent_id: "agent_003", role: "开发工程师" }
  ]
}

// 智能体任务队列
{
  agent_id: "agent_002",
  queue: [
    {
      id: "task_001",
      from: "agent_001", // 组长
      type: "team_task",
      content: "设计系统架构图",
      status: "pending", // pending, processing, completed, failed
      created_at: "2024-01-15T10:05:00Z"
    }
  ]
}
```

---

**开发优先级建议**：
1. 后端：`agent_service`扩展、消息路由系统
2. 前端：双面板布局、小组管理界面
3. 数据库：新增`teams`、`team_messages`表
4. 联调：任务队列逻辑、实时通信
